What is M2?

M2 is a method used in 3BLD to solve the 12 edge pieces using a specific algorithm that targets the middle slice (the "M-slice"). It’s paired with a fixed buffer, typically DF (U in Speffz).

Why use M2?

  • It’s efficient and consistent for edge solving
  • All targets can be handled with simple setup moves + a core M2 algorithm
  • Pairs well with 3-style or Old Pochmann for corners

    • M2 Algorithm

    The main algorithm used in M2 is M2 (swapping algorithm)

    Buffer and Target

    The buffer used in M2 is typically the U sticker in Speffz. The goal is to cycle the sticker at DF with a target edge, one at a time.

    Example Setup Moves

    Here are some setup moves from the buffer U to various edge stickers. This assumes sticker-level targeting (not piece) in Speffz.

  • A: M2 (no setup moves)
  • UL: U2 ... M2 ... U2
  • UR: U' ... M2 ... U
  • FR: R ... M2 ... R'
  • FL: L' ... M2 ... L
  • BD: D2 ... M2 ... D2

    • Tip: Make a chart or flashcards to drill these setups!

    Special Cases

  • Flipped buffer: Skip, it's solved.
  • Parity (odd number of targets): Add an extra letter to balance it out (e.g., use a dead sticker).
