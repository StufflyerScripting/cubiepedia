Getting Started

Solving a Rubik's Cube without seeing it may sound extreme—but it’s not only possible, it’s fun! You’ll just need to learn a few techniques, a couple of algorithms, and how to memorize piece positions.

Requirements:

  • Know how to solve a cube normally
  • Learn 3 algorithms
  • Be ready to memorize 20–30 letters per solve

Step-by-Step Plan

  1. Figure out where each piece should go
  2. Move one piece at a time using swap algorithms
  3. Use setup moves to bring the right piece into position

Edge Solving

Keep white on top and green in front. You’ll swap pieces using this algorithm:

Edge Swap Algorithm:

R U R' U' R' F R2 U' R' U' R U R' F'

This swaps UR (buffer) with UL (target).

Setup Examples

  • To swap with FL: L' → algorithm → L
  • To swap with LF: Dw' L → algorithm → L' Dw

Important: Focus on the buffer (UR) to determine what to swap. Incorrect targets = flipped pieces.

Setup Tips

  • Don’t use R, U, F, B for setups
  • Only use L, Lw, Dw
  • D face: D + L2 setup
  • E layer: Dw + L or L'
  • M layer: Lw to D or U face, then continue

Corner Solving

Same orientation: white on top, green in front.

Corner Swap Algorithm:

R U' R' U' R U R' F' R U R' U' R' F R

This swaps LBU (buffer) with DFR (target).

Setup Examples

  • To swap with FRU: R' → algorithm → R
  • To swap with UFR: F2 D → algorithm → D' F2

Setup Tips

  • Only use D, F, R for setup
  • Place the target at DFR using 1–2 moves max

Memorization

Use a lettering system like Speffz. Assign each sticker a letter, and memorize a sequence of letters representing the target stickers of each swap.

Cycle Example (Edges)

  1. UR (buffer) → Q
  2. Q → U
  3. U → S
  4. S → R (back to buffer)

Continue with new cycles or flipped edges if needed.

Flipped Edges

Choose either sticker and memorize its position. Example: D → E.

Corner Memorization

Same idea as edges. Track the piece sticker-by-sticker and form pairs.

Visualization Tips

Turn your letter pairs into images or a story. For example:

QU → "Quick", SR → "Star", NX → "Nexus", etc.

Parity

If both edge and corner memorization have an odd number of letters, you’ll need to run a parity algorithm before solving corners.

Parity Algorithm:

R U' R' U' R U R D R' U' R D' R' U2 R' U'

Solving Process Summary

  1. Edge memo → solve edges
  2. Run parity if needed
  3. Corner memo → solve corners
  4. Each letter: setup → algorithm → undo

Don’t think about cycles while solving. Follow your memorized letters and the cube will be solved.

Example Solve

Scramble:

B2 R2 B2 L U2 R' B2 R2 B2 R' U' B' F L' B D R B2 D B' D' U

Edges:

QU SR NX IV PR DE

Corners:

MW QX MD UI VP

No parity needed (even + even).
